# NexusAI API Contract

All endpoints are hosted at `/api/v1/` and consume/produce `application/json`.

## Agent Endpoints

### `POST /agent/query`
Initiates a new agent search based on user input.
- **Request Body**:
  ```json
  {
    "user_id": "uuid",
    "query": "I need 3 software developers in London with Python experience."
  }
  ```
- **Response (200 OK)**:
  ```json
  {
    "search_id": "uuid",
    "status": "awaiting_approval",
    "recommendations": [...],
    "draft_action": {
      "type": "email",
      "content": "..."
    },
    "execution_trace": [
      {"step": "parse", "status": "success"},
      {"step": "search", "status": "success"},
      {"step": "validate", "status": "success"}
    ]
  }
  ```

### `POST /agent/approve`
Approves a pending action generated by the agent.
- **Request Body**:
  ```json
  {
    "search_id": "uuid",
    "action_type": "email",
    "payload": {"to": "...", "body": "..."}
  }
  ```
- **Response**: `{"status": "approved", "timestamp": "..."}`

### `POST /agent/reject`
Rejects a pending action.
- **Request Body**:
  ```json
  {
    "search_id": "uuid",
    "reason": "Not enough budget"
  }
  ```

## Data Endpoints (CRUD)

### `GET /suppliers`
Lists suppliers with pagination and basic filtering.
- **Query Params**: `limit`, `offset`, `location`, `certification`

### `GET /suppliers/{id}`
Retrieves deep details of a specific supplier.

### `GET /professionals`
Lists professionals.

### `GET /logs/{search_id}`
Retrieves the complete execution and validation trace for a specific query, used by the frontend LangSmith-style UI.
